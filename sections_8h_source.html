1 /*
2  * section | speed | execute | shared | cached | buffered | DMA | ISR
3  * ----------|--------|---------|--------|--------|----------|-----|----
4  * FASTDATA | Fast | No | No | No | No | Yes | Yes
5  * FASTEXEC | Fast | Yes | No | No | No | No | Yes
6  * SHAREDATA | Normal | No | Yes | No | No | Yes | Yes
7  * ROMEM | Normal | No | Yes | Likely | Likely | Yes | Yes
8  * <default> | Normal | Yes | Yes | Likely | Likely | No | No
9  *
10  * Memory sections should be created based on each sections capabilities,
11  * specifically, with attention to:
12  * - DMA R/W safety without any special cache coherence effort
13  * - ISR R/W safety without any special cache coherence effort
14  *
15  * If the platform needs no special considerations for any of these,
16  * the names should still be defined but the content can be blank.
17  *
18  * Applications and libraries should prepend these for each individual
19  * variable / function :
20  *
21  * - Variable R/W in ISR + DMA + Cortex contexts : SHAREDATA
22  * - Variable R/W in DMA + Cortex contexts only : FASTDATA / SHAREDATA
23  * - Variable R/W in ISR + Cortex contexts only : FASTDATA / SHAREDATA
24  * - Variable R/W in Cortex context only, high frequency : FASTDATA
25  * - Variable R/W in Cortex context only, low frequency : <default>
26  *
27  * - Function executes in ISR context : FASTEXEC
28  * - Function executes in Cortex context only, high frequency : FASTEXEC
29  * - Function executes in Cortex context only, low frequency : <default>
30  *
31  * - Constant data stored in flash : ROMEM (to migrate from romem.h)
32  *
33  * These are tentative definitions which might need additional review. The
34  * following open questions must be addressed:
35  *
36  * - It is unclear whether DMA can always access FASTDATA
37  * - It is unclear whether FASTDATA is ok for data accessed in DMA
38  * and ISR contexts
39  *
40  * The implementation of this is as follows:
41  *
42  * STM32H7RS
43  * ---------
44  * - FASTDATA is DTCM. This is non-cached, non-shared core coupled memory.
45  * It can be accessed from an interrupt context and probably can be
46  * accessed from a DMA context, though there seem to be conflicting
47  * information about this. Variables should be treated as volatile
48  * when being accessed from the Cortex, if the DMA / Interrupt are
49  * expected to write to them. (.dtcm)
50  *
51  * - FASTEXEC is ITCM. This non-cached, non-shared core coupled memory. It
52  * can be used in an interrupt context. (.itcm)
53  *
54  * - SHAREDATA is non-cached, shared regular RAM in a separate section.
55  * The linker file should put this in a memory region where prefetch
56  * and caching are disbaled in the MPU. It can perhaps also be placed
57  * into the 16kb SRAM1/2 if it needs to be on AHB instead of AXI. This
58  * could perhaps be used to store data which is handled by the DMA and
59  * within interrupt contexts, when the data is large but slow enough
60  * or where DTCM might hit other constraints. (.shared)
61  *
62  * - Default is cacheable RAM and FLASH. ICACHE and DCACHE are presently
63  * not enabled, however, expect this to be enabled at some point.
64  *
65  * - ROMEM not implemented.
66  *
67  * - It seems HPDMA is suited to DTCM and the large SRAM1/2/3/4 on AXI,
68  * while GPDMA is suited to the 16kb SRAM1/2 and AHB peripherals.
69  * Details to be worked out. It may be necessary to add an additional
70  * section to be able to use this. For the moment, plan is to use the
71  * 16kb SRAM1/2 for GPDMA use with SHAREDATA, while HPDMA can be used
72  * with FASTDATA in DTCM.
73  *
74  */
